## Conventions

### Frontend
- Standalone Angular components (no NgModules)
- Tailwind CSS v4 with utility-first approach
- TypeScript strict mode enabled
- Component naming: PascalCase + `.component.ts`
- Container/Presentational pattern for state management

### Backend
- NestJS REST API (already exists)
- Dependency injection via NestJS modules
- DTO validation with class-validator

### Testing
- E2E: Playwright with Page Objects
- Unit: Jest with test files co-located

### Code Quality
- ESLint + Prettier for formatting
- Conventional commits (no AI attribution)
- Type-safe everywhere (strict TypeScript)
